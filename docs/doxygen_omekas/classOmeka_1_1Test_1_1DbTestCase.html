Detailed Description

Database test case.

Tests that need access to Doctrine's entity manager (e.g. for writing to and querying the database) should extend off this class. For every test, this starts a transaction during setUp, and rolls back any changes to the database during tearDown. A fresh database should have been installed when bootstrapping PHPUnit.
